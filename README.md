# lean_phase1

This repo is for learning Lean 4 theorem prover and the basic structure of programmatically formalized proofs. 

It samples different proof styles, tactics, and problem types.

## Challenges so far:
- The more "foundational" tactics like `intro`, `apply`, and `exact` are harder to use than higher-level ones like `rw`, `rfl`, and `simp` because they formalize steps that are easy to overlook when doing proofs by hand.

## Tools:
- The Infoview is a very helpful tool for tracking current proof progress and understanding what each step achieves.

## Next steps:
- Work through more complex proofs and understand how to structure proofs with more moving parts.
